# Politique de Confidentialité

**Dernière mise à jour :** 21 avril 2026

## 1. Introduction

CheckIt (« l'Application ») est une application mobile de gestion de listes d'épicerie développée par Olivier Allard-Jacquin. Cette Politique de Confidentialité explique comment nous collectons, utilisons, stockons et protégeons vos données personnelles.

En utilisant CheckIt, vous acceptez les pratiques décrites dans cette politique. Si vous n'acceptez pas ces conditions, veuillez ne pas utiliser l'Application.

---

## 2. Données Collectées

### 2.1 Données d'Authentification
- Adresse e-mail et mot de passe (via Firebase Authentication)
- Identifiant utilisateur unique

### 2.2 Données de Contenu
- Listes d'épicerie créées
- Produits ajoutés aux listes (nom, marque, description, quantité, unité)
- Sections/rayons personnalisés
- Statut des articles (coché/non-coché)
- Historique des produits

### 2.3 Données de Partage
- Adresses e-mail des utilisateurs avec lesquels vous partagez des listes
- Permissions d'accès aux listes partagées

### 2.4 Données Techniques
- Type et version du système d'exploitation
- Modèle d'appareil
- Identifiants publicitaires (si applicables)
- Informations de session

### 2.5 Données Non Collectées
- Localisation GPS
- Contacts ou calendrier
- Données de santé
- Données biométriques
- Historique de navigation

---

## 3. Utilisation des Données

Nous utilisons vos données pour :

- **Fourniture du service** : Créer, stocker et synchroniser vos listes d'épicerie
- **Authentification** : Sécuriser votre compte et vérifier votre identité
- **Partage** : Permettre la collaboration sur les listes avec d'autres utilisateurs
- **Amélioration** : Analyser l'utilisation pour améliorer l'Application
- **Communication** : Vous envoyer des notifications essentielles ou des mises à jour
- **Sécurité** : Détecter et prévenir les activités frauduleuses ou abusives

---

## 4. Stockage et Hébergement des Données

### 4.1 Stockage Cloud
Vos données sont stockées dans **Firebase Firestore** (Google Cloud), avec chiffrement au repos et en transit (HTTPS/TLS).

### 4.2 Stockage Local
Les données sont également synchronisées localement sur votre appareil via `AsyncStorage` pour un accès hors ligne.

### 4.3 Localisation des Données
Les serveurs Firebase peuvent être situés à :
- États-Unis
- Europe (région UE pour les données UE)
- Autres régions selon la configuration Google Cloud

Consultez la [Politique de Confidentialité de Google](https://policies.google.com/privacy) pour plus de détails.

---

## 5. Partage des Données

### 5.1 Partage Contrôlé
- Vous contrôlez entièrement qui a accès à vos listes via la fonction de partage
- Les données ne sont partagées que avec les utilisateurs que vous autorisez explicitement

### 5.2 Pas de Vente de Données
Nous ne vendons, n'échangeons ni ne louons jamais vos données personnelles à des tiers.

### 5.3 Tiers de Confiance
- **Firebase (Google)** : Hébergement et authentification
- **Open Food Facts** : Données de codes-barres (requêtes anonymes uniquement)

---

## 6. Droits de l'Utilisateur

Conformément au RGPD et autres lois sur la confidentialité, vous avez le droit de :

- **Accès** : Demander l'accès à vos données personnelles
- **Rectification** : Corriger des données inexactes
- **Suppression** : Demander la suppression de votre compte et de vos données
- **Portabilité** : Recevoir une copie de vos données dans un format structuré
- **Opposition** : Vous opposer à certains traitements

Pour exercer ces droits, contactez : **oallardjacquin@gmail.com**

### 6.1 Suppression du Compte
Lors de la suppression de votre compte, toutes vos données seront supprimées dans les 30 jours, sauf si la loi exige leur conservation.

## 7. Sécurité

Nous mettons en place des mesures de sécurité pour protéger vos données :

- Chiffrement HTTPS/TLS pour les communications
- Chiffrement au repos dans Firebase Firestore
- Authentification sécurisée via Firebase Auth
- Contrôle d'accès basé sur les rôles
- Audit des logs d'accès

**Limitation de responsabilité** : Bien que nous fassions tout notre possible pour sécuriser vos données, aucune transmission sur Internet n'est 100 % sûre. Utilisez l'Application à vos propres risques.

---

## 8. Durée de Conservation

- **Données de compte actif** : Conservées tant que votre compte est actif
- **Données après suppression du compte** : Supprimées dans les 30 jours (sauf obligations légales)
- **Logs de serveur** : Retenus pendant 90 jours pour la sécurité et le débogage
- **Sauvegardes** : Peuvent être conservées pendant une période supplémentaire selon les politiques de Google Cloud

---

## 9. Enfants et Mineurs

CheckIt n'est pas destinée aux enfants de moins de 13 ans. Nous ne collectons pas sciemment de données d'enfants. Si nous découvrons que des données d'enfant ont été collectées, nous les supprimerons immédiatement.

Pour les utilisateurs entre 13 et 18 ans, le consentement parental est recommandé.

## 10. Cookies et Technologies de Suivi

CheckIt n'utilise pas de cookies. Cependant :
- Firebase peut utiliser des identifiants pour la sécurité et l'authentification
- Les logs serveur peuvent inclure des adresses IP
